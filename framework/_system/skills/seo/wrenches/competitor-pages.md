# seo-competitor-pages — vs / alternatives pages

"Vs" and "alternatives to" pages are high-intent SEO real estate. Searchers are evaluating purchases. This wrench plans + structures these pages for SEO + conversion.

---

## When to fire

- "Build a vs page for X vs Y"
- "Alternatives to Z page"
- "Comparison page"
- Competition-heavy SaaS or e-commerce site

Don't fire when:
- No competitive context (no comparison to make)
- Pure feature page (route to design-studio)

---

## Page types

| Type | Use |
|---|---|
| **Direct comparison** | the user's product vs one specific competitor (head-to-head) |
| **Alternatives** | "Alternatives to <competitor>" — the user's product is one of several listed |
| **Roundup** | "Best X for Y" — feature multiple options including the user's |
| **Migration guide** | "How to switch from X to the user's product" |

---

## Page structure (proven shape)

```
1. Hero: clear claim ("the user's product is the X for Y use case")
2. Quick comparison table (3-5 key dimensions, scannable)
3. When the user's product is better (specific use cases)
4. When competitor X is better (be honest — credibility)
5. Feature-by-feature matrix (depth)
6. Pricing comparison
7. Migration / switch guide (if applicable)
8. FAQ (target related queries)
9. CTA to try the user's product
```

The "when competitor X is better" section is counterintuitive but increases trust + conversion. AI tools also cite balanced comparisons more.

---

## Schema

Use ItemList + Product schema for the comparison structure. AI Overviews increasingly surface structured comparisons.

---

## Conversion optimization

- Above-the-fold: clear claim + scannable table + CTA
- Honest balance increases trust
- "Try" CTAs > "Learn more" CTAs (specific action)
- Social proof near CTAs (review counts, testimonials)
- Direct migration path if applicable
